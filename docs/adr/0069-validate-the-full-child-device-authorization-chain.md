# Validate the full Child Device authorization chain

Every authenticated Child Device HTTP request will treat the JWT's credential, Active Enrollment, and Child Device IDs only as claims, then validate the current active credential, enrollment, device, Child Profile, and Household records and their relationships before application work. Child Profile and Household identity are derived from those records rather than accepted from the request or JWT, and protected internal queries and mutations revalidate the chain within their own transaction to prevent revocation races. This adds bounded direct lookups to low-frequency Child Device operations but makes credential revocation, device replacement, enrollment termination, and domain deletion effective without waiting for JWT expiry.
