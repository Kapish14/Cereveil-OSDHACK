# Limit scam detection to selected active apps

Scam Text Detection will use only individual visible, non-editable accessibility text nodes from Guardian-selected Monitored Apps, debounced for two seconds per foreground package and requiring at least 20 normalized characters. Cereveil will not concatenate a whole screen or conversation and excludes editable drafts, password fields, Cereveil overlays, obvious timestamps, and app chrome. It will not use OCR, analyze notification content, access background message databases, or log or persist candidate text; the tokenizer may truncate a candidate to its existing 128-token limit. The `standard` development sensitivity preserves Enfold's existing decision rule: one of fraud classes 2–7 must be the top-ranked class, with no additional confidence floor; alternate sensitivities remain development tuning that must be validated separately. This accepts that unopened, muted, hidden, image-based, inaccessible, or unusually fragmented content will not be detected.
