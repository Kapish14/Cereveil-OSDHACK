# Detect NSFW content from active-window screenshots

NSFW Screen Detection will capture temporary screenshots only while Guardian-selected Monitored Apps are active on an unlocked Child Device, run the CNN locally, and discard captured pixels without upload or persistence. Cereveil starts from Revive's node-bounded image-region pipeline, but deliberately adds a fallback for video, canvas, or custom-rendered content that exposes no usable image nodes: it classifies only the Monitored App window and, on a positive result, blurs that window. In split-screen or picture-in-picture, classification and blur remain confined to regions belonging to the Monitored App; Cereveil never includes system bars, keyboards, overlays, or another app's window. Detection stops when the screen turns off, the device locks, the monitored window disappears, or Active Screen Safety is disabled. Capture and overlay tuning starts from the proven Revive configuration and remains app-owned rather than Guardian-configurable. The `standard` development sensitivity preserves Revive's existing 60% sensitivity mapping to a `0.40` NSFW confidence threshold; alternate sensitivities remain development tuning that must be validated separately.
