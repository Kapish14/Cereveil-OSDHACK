# Coordinate Remote Audio through Convex state and WebRTC signaling

Cereveil will coordinate Remote Audio through Convex authorization, request/session state, Child Device commands, and short-lived WebRTC signaling rows, while the live audio stream itself travels through WebRTC. Remote Audio remains an on-demand, Child-confirmed capability outside the versioned Supervision Policy; availability derives from active supervision, current health and capabilities, notification delivery, live request state, and cooldown. Guardian Mode may create a Remote Audio Request only for an active Child Device enrollment, outside cooldown, and when no live request already exists. Creation uses a client operation ID, returns the initiating Guardian Device's existing request on retry or repeated tap, and rejects another Guardian Device as busy rather than allowing takeover. Each request is bound to its initiating Guardian Device: only that device may read or publish WebRTC signaling and receive audio, while any active Guardian Device on the same Guardian Account may safely terminate it. Child Mode must fetch the command and show a persistent notification with Start audio and Decline actions. Only the Child's Start audio interaction launches the microphone foreground service; after the foreground notice is active and microphone readiness is verified, Child Mode creates the SDP offer. The initiating Guardian Device publishes the SDP answer, and those two endpoints trickle ICE candidates through Convex. Every stop path closes local capture or playback and the peer connection before attempting its terminal Convex mutation; mutation failure never causes local media to resume, and the backend still removes the request at its fixed deadline. Phone-call interruption, audio-focus loss, microphone contention, and audio-route failure are immediately terminal. WebRTC `DISCONNECTED` may recover to `CONNECTED` within five seconds while the Child notice remains visible and the fixed deadline continues; `FAILED`, `CLOSED`, or five seconds without recovery terminates the request, and v1 performs no ICE restart. Convex then applies cooldown and removes request/session and signaling state. The initial implementation uses STUN-only WebRTC and accepts failures on restrictive networks until TURN support is added later.
