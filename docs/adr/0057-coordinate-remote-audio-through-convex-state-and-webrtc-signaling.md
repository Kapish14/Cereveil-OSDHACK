# Coordinate Remote Audio through Convex state and WebRTC signaling

Cereveil will coordinate Remote Audio through Convex authorization, session state, Child Device commands, and short-lived WebRTC signaling rows, while the live audio stream itself travels through WebRTC. Guardian Mode may request Remote Audio only for an active Child Device enrollment, outside cooldown, and when no active session already exists. Child Mode must fetch the command, show a persistent Child-facing notice immediately, verify microphone/foreground-service readiness, exchange offer/answer/ICE signaling through Convex, and stream live audio without recording. Sessions end when the Guardian stops, the Child stops, the time limit is reached, WebRTC fails, or required permissions/network become unavailable; Convex then marks the lifecycle state, applies cooldown, and expires signaling data. The initial implementation uses STUN-only WebRTC and accepts failures on restrictive networks until TURN support is added later.
