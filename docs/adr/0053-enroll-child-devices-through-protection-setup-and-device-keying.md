# Enroll Child Devices through Protection Setup and device keying

Cereveil will activate Child Device enrollment only after the Child app completes Protection Setup, scans a short-lived single-use Enrollment Code, and proves device-specific key material generated in Android Keystore. The Convex Device Identity module will validate the Enrollment Code, ensure the Child Profile has no active enrollment, create the Child Device, Active Enrollment, Child Device Credential, and policy application state, then issue a short-lived Child Device JWT. The Enrollment Code is only a bootstrap credential; ongoing access uses the revocable Child Device Credential and short-lived JWTs. This keeps protection readiness, role lock, device identity, and backend authorization aligned at the moment supervision becomes active.
