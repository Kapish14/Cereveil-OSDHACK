# Create Safety Alerts from on-device incidents and weekly backend summaries

Cereveil will create Safety Alerts only from Child Mode's on-device safety detections. Child Mode uses a short-lived local suppression fingerprint to avoid duplicate alerts for the same visible incident, shows the Child-facing Safety Warning locally, and sends only metadata to Convex. Convex verifies the ChildDeviceActor, Active Enrollment, and applied policy before storing the Safety Alert, creating a Guardian Notice, and using FCM as a wake-up signal. Weekly backend scheduled work aggregates retained Safety Alerts into a Safety Incident Summary, notifies Guardians, deletes the individual alerts covered by the summary, and removes the previous summary before or when the new one replaces it. This preserves immediate Guardian awareness without retaining raw content or long-term incident history.
