# Version strongly typed Supervision Policy schemas

Each Supervision Policy snapshot will declare a schema version and contain explicitly validated, strongly typed feature sections rather than an arbitrary feature-name-to-JSON map. Adding a feature extends the policy schema deliberately; historical versions remain unchanged and missing sections in older schemas retain documented safe meanings, while the next Guardian change creates a complete snapshot in the current schema. Phase 9 introduces schema v3 with independent Scam Text Detection and NSFW Screen Detection sections, each containing `enabled`, its own Monitored App package set, and a `lower`, `standard`, or `higher` sensitivity; v1 and v2 retain disabled, empty, `standard` defaults for both. Child Mode reports its maximum supported policy schema during enrollment and authenticated health reporting, and Convex rejects a Guardian change requiring a newer schema than the Active Enrollment supports. This preserves immutable history, prevents unsupported policies from becoming permanently desired, and allows Convex and Child Mode to reject malformed or unsupported configuration instead of interpreting unvalidated or incomplete data.
