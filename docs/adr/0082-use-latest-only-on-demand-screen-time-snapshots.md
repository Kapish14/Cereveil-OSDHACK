# Use latest-only on-demand Screen Time Snapshots

While the Screen Time policy feature is enabled, Cereveil will expose only a latest Screen Time Snapshot containing Android-reported, today-so-far foreground duration per app, with no Cereveil-derived session counts, raw usage events, daily or weekly summaries, or history. Enabling the feature authorizes retrieval from local midnight through the measurement time, including usage from before the policy was applied or the Child Device was enrolled on that same day; disabling immediately deletes the snapshot, but re-enabling on the same day again authorizes the full midnight-to-now total, including the disabled interval. Guardian Mode must disclose this behavior before enablement or re-enablement. Child transparency comes from Protection Setup and the persistent Child Mode status surface rather than a new notification on every enablement or refresh. When the selected Child Profile's Screen Time surface becomes visible and its snapshot is at least two minutes old, Convex creates one deduplicated bounded Screen Time Refresh Request and uses normal-priority FCM to wake that Online Child Device with Usage Access; Child Mode queries Android, uploads the current per-app totals, and atomically replaces the previous snapshot. Child Mode does not queue a snapshot whose request cannot complete; partial staging rows expire and a later request measures again. Opening Guardian Mode does not refresh every Child in a Household. A snapshot becomes invalid at the Child Device's next local midnight and is deleted rather than becoming a prior-day record. This supersedes ADR-0017, ADR-0018, and ADR-0049, favoring immediately useful current state and simpler retention over historical trends.
