# Start Remote Audio with STUN-only WebRTC

Cereveil's initial Remote Audio implementation will use WebRTC with STUN-only NAT traversal and no TURN relay provider. Convex remains responsible for Remote Audio authorization, request/session state, signaling, cooldown checks, and lifecycle cleanup, while WebRTC carries the live audio stream directly between devices when connectivity permits. Authorized request state supplies a backend-configured list of STUN URLs: development and hackathon builds may default to `stun:stun.l.google.com:19302`, while production must explicitly configure its STUN service instead of silently depending on that public endpoint. No TURN credentials or fallback are exposed in v1. This avoids adding a paid relay dependency during the hackathon phase, but accepts that Remote Audio may fail on restrictive networks until TURN relay support is added later.
