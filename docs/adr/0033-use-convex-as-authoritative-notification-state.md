# Use Convex as authoritative notification state

Cereveil will treat Convex as the authoritative source for Guardian Notices, Child Device commands, delivery state, deduplication, and acknowledgements. FCM will be used only as a best-effort delivery and wake-up transport: push payloads contain minimal non-sensitive metadata such as event ID, type, timestamp, and generic display text, and every device reconciles the authoritative state from Convex before acting on missed, duplicated, or out-of-order events. Safety Alert lock-screen text does not reveal the detection type, app, or confidence band; those details appear only after opening authenticated Guardian Mode. High-priority FCM is reserved for immediate user-visible events such as Safety Alerts, Access Requests, Tamper Alerts, and Remote Audio requests. This accepts extra backend state and acknowledgement logic, but avoids relying on Android push delivery guarantees and keeps sensitive supervision details out of third-party notification payloads.
