# Structure Convex backend around function wrappers and feature modules

Cereveil will structure its Convex backend around feature modules and shared function wrappers rather than copying a classic controller-service-repository stack directly. Public Convex queries, mutations, actions, and HTTP actions are the backend entrypoints; shared wrappers such as guardianMutation, childDeviceMutation, and publicHttpAction will create request context, resolve actors, enforce authentication, map safe errors, and emit privacy-safe logs. Feature modules will keep public entrypoints, use-case logic, focused data helpers, and validators close together, while external side effects such as FCM are performed through scheduled/internal actions rather than directly inside business mutations. This adds a small backend framework layer, but keeps authorization, error handling, logging, and module ownership consistent across the app.
