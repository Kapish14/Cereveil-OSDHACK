# Make remote audio Child-confirmed and child-stoppable

A Guardian creates a Remote Audio Request that immediately shows the Child an ongoing notification with Start audio and Decline actions. Because Cereveil targets Android API 36 and `RECORD_AUDIO` is a while-in-use permission, Child interaction with Start audio is required before Child Mode launches the microphone foreground service; no microphone capture or WebRTC offer occurs before that interaction. The request may be visible while the device is locked, and Decline remains immediately usable, but Start audio requires device unlock so another person cannot initiate capture from a locked Child Device. Once started, the notification becomes a non-dismissible active Remote Audio notice whose Stop action remains immediately available without authentication throughout the session. Child Mode mirrors the request or session status and controls while its UI is open, but does not cover other apps with an accessibility overlay. It must terminate without capture if Android cannot present the notification or launch the service. The backend-owned deadline remains two minutes after `requestedAt`, covering time awaiting the Child, connecting, and active media, so delayed interaction or signaling never extends the request. Decline, failure, expiry, Guardian stop, or Child stop terminates the request and begins the three-minute cooldown. This preserves a bounded emergency check-in flow within modern Android's microphone restrictions while making capture explicitly Child-confirmed, visible, and terminable.
