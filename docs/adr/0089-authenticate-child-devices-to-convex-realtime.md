# Authenticate Child Devices to Convex realtime

Cereveil will replace its HTTP-only HS256 Child Device JWT with a short-lived ES256 custom JWT carrying a key ID, subject, URL issuer, audience, and the existing credential, Active Enrollment, and Child Device claims. Device Identity will sign with a dedicated backend-held P-256 private key that is distinct from every Child Device's Keystore proof-of-possession key, expose only its public key through JWKS, and register the issuer as a Convex `customJwt` provider. This allows Child Mode to use an authenticated Convex Android client for realtime Remote Audio state and WebRTC signaling. Enrollment and proof-of-possession token refresh remain HTTP endpoints, and every HTTP or realtime function must validate the full Child Device authorization chain. Because Cereveil has no deployed Child Device users or legacy state, the implementation will make a clean cutover: remove HS256 issuance and verification and switch all Child Device request authentication to ES256 together, without a dual-token migration window. This adds asymmetric key management and a realtime Child client but avoids high-frequency HTTP polling during live signaling.
