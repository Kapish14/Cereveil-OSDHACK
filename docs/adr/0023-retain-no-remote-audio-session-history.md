# Retain no remote-audio session history

After a Remote Audio Request ends, Cereveil will delete its request, signaling rows, typed Child Device Command, and associated FCM delivery-attempt rows, retain neither audio nor session metadata, and provide no session-history view, mirroring the decision not to retain location history. Remote Audio commands bypass the normal command-retention period. The sole per-Child exception is a minimal Remote Audio Cooldown row keyed by Child Profile and containing only `cooldownUntil`; it carries no request reference, initiator, timestamps describing the ended request, or outcome, and is deleted when the three-minute cooldown expires. Observability is limited to anonymous aggregate counters without per-request identifiers, actors, timestamps, durations, or payloads. This minimizes sensitive records but deliberately means neither the Guardian nor Child can later verify which Guardian Device initiated a request or when it occurred.
