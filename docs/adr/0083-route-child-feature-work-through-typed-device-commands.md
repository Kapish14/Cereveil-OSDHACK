# Route Child feature work through typed device commands

Cereveil will extend the existing Child Device Command lifecycle with strongly typed `refresh_location`, `refresh_screen_time`, `reconcile_access_grants`, and `request_remote_audio` variants alongside `apply_policy_version`. Feature modules create authoritative domain records first and commands reference those records without embedding location, Screen Time totals, Access Grant authority, Remote Audio state, or full policy state; FCM remains a generic `child_command` wake-up, after which Child Mode reconciles commands and processes them idempotently. A `request_remote_audio` command shares the referenced Remote Audio Request's expiry and is acknowledged only after its Start audio/Decline notification is successfully presented; failure to present terminates the request before the command is marked handled, while an already absent or expired request is acknowledged without UI. Unlike ordinary commands, it is deleted with its FCM delivery-attempt rows on every terminal Remote Audio transition and never enters the normal retained terminal-command lifecycle. This centralizes expiry, rejection, retry, and delivery behavior while keeping each feature's state and authorization in its owning module.
