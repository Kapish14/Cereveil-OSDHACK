# Use fifteen-minute Child Device JWTs

Child Device JWTs will live for fifteen minutes in v1 and use a dedicated backend ES256 signing key so they can authenticate both Device Identity HTTP requests and authorized realtime Convex operations. They carry the Child Device Credential, Active Enrollment, and Child Device identity, while sensitive backend operations still resolve current credential and enrollment state instead of relying only on token expiry. This keeps bearer-token exposure bounded after revocation, avoids excessive Keystore challenge-signing refresh traffic, and supports short realtime Remote Audio signaling without weakening authorization.
