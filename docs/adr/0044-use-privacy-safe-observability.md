# Use privacy-safe observability

Cereveil will collect operational logs, metrics, and crash diagnostics only when they are safe for a child-supervision product. Logs may include sync outcomes, policy version application, FCM token registration status, heartbeat receipt, capability changes, Remote Audio lifecycle events, Safety Alert IDs, and scheduled cleanup outcomes. Logs, analytics, crash reports, debug traces, and support exports must not contain raw scam text, screenshots, OCR text, audio, exact location history, full notification contents, accessibility node contents, or Child legal/contact identity data. This may make debugging harder, but prevents observability systems from becoming an accidental store of sensitive child content.
