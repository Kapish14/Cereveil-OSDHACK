# Resolve Access Requests through authoritative backend grants

Cereveil will process Access Requests through Convex as authoritative state. When Child Mode is online and a blocked app is visible, it may create an Access Request only if the app is actually blocked under the applied policy; Convex then creates a Guardian Notice and uses FCM only to wake Guardian Devices. If Child Mode is offline, it shows that the Guardian is unreachable and does not queue the request. Guardian approval changes the Access Request to approved, creates an Access Grant, and wakes the Child Device to fetch the grant; denial changes the request to denied without creating a grant. This keeps temporary access decisions backend-authorized and prevents offline or stale requests from silently appearing later.
