# Create Safety Alerts from on-device incidents

Cereveil will create Safety Alerts only from Child Mode's on-device safety detections. Child Mode uses a ten-minute memory-only local suppression fingerprint to avoid duplicate alerts for the same visible incident, shows the Child-facing Safety Intervention locally, and sends only metadata to Convex. Fingerprints are cleared on process death, restart, Active Screen Safety disablement, or End Supervision; Cereveil accepts one conservative repeated incident after a restart rather than persisting content-derived fingerprints. While offline it queues only idempotent metadata for at most seven days and never queues raw content or fingerprints. Convex verifies the ChildDeviceActor, Active Enrollment, and applied policy before storing each Safety Alert. It creates an immediate Guardian Notice only when the alert arrives within five minutes of detection and the fixed, non-sliding two-minute cooldown keyed by Child Profile and detection type permits; that one backend decision applies to every active Guardian Device, while other retained alerts remain available without a delayed notice. Individual Safety Alerts expire after one week and are not aggregated into summaries.
