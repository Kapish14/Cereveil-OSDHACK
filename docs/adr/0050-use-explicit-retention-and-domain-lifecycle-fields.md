# Use explicit retention and domain lifecycle fields

Cereveil will model row lifecycle explicitly with fields such as createdAt, updatedAt, expiresAt, revokedAt, endedAt, and status rather than relying on implicit cleanup behavior. Expiring rows such as Safety Alerts, Screen Time Summaries, Access Requests, Access Grants, commands, signals, and live sessions will include expiresAt so backend scheduled cleanup can delete or expire them predictably. Long-lived identity and ownership rows such as Guardian Accounts, Households, Child Profiles, devices, enrollments, and credentials will use status and lifecycle timestamps instead of pretending they expire naturally. This adds consistency work to the schema, but makes privacy retention, revocation, deletion, and UI validity rules explicit.
