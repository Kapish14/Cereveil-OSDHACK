# Activate enrollment only after protection setup and pairing

Protection Setup must complete before Child Mode permits QR pairing. Permissions may be granted on a Prepared Child Device, but monitoring and data collection begin only after the Enrollment Code successfully binds it to a Child Profile and creates an Active Enrollment. Setup may be paused, resumed, or cancelled rather than forcing a system permission; later capability revocation keeps the enrollment active in Protection Degraded state and emits a Tamper Alert.
