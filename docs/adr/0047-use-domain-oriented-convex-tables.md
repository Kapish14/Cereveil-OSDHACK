# Use domain-oriented Convex tables

Cereveil will start with domain-oriented Convex tables for Guardian identity, Household ownership, Child Profiles, active Child Device enrollment, revocable Child Device credentials, versioned Supervision Policies, policy application state, notices, device commands, access requests and grants, safety alerts and summaries, screen-time summaries, location state, live location sessions, Remote Audio sessions, FCM tokens, and audit events. Push tokens are modeled separately from device identity, and Child Device credentials are modeled separately from Active Enrollment. This creates more tables than a minimal prototype, but preserves the distinctions needed for authorization, revocation, retention, offline enforcement, and notification delivery.
