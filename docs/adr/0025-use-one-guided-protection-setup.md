# Use one guided protection setup

Before pairing, Child Mode will present one Guardian-completed Protection Setup checklist that requests all capabilities required by the current build in sequence. Each capability retains its own concise disclosure and Android system interaction, satisfying platform requirements while avoiding fragmented setup later. The hackathon build requests Accessibility, Usage Access, background location, microphone, permission to display Cereveil notifications, and battery-optimization exemption up front; it does not request Android Notification Listener access and omits VPN access because ADR-0029 defers Safe Browsing. A later production build that includes Safe Browsing will add the VPN step. Completing the checklist creates a Prepared Child Device but must not start monitoring or collecting data before QR pairing succeeds.
